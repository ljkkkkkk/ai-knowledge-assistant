# 第06课：配置管理和环境变量

## 这一课要解决什么

上一课我们已经让 FastAPI 服务可以启动，并且通过 `/health` 返回健康状态。

但 `app/main.py` 里还有一个工程问题：

```text
项目名称、版本号、运行环境这些信息都直接写在代码里。
```

小项目这样可以跑，大一点的项目就会很难维护。

比如以后你可能会有：

```text
开发环境 development
测试环境 testing
生产环境 production
数据库地址
大模型 API Key
模型名称
```

这些东西不应该散落在业务代码里，而应该统一放进配置层。

## 什么是配置管理

大白话说，配置管理就是：

```text
代码负责逻辑，配置负责变化。
```

代码里写：

```text
我要读取数据库。
```

配置里写：

```text
数据库在哪里、用户名是什么、当前用哪个模型。
```

这样以后换环境时，不用改代码，只改配置。

## 新增的 `app/core/config.py`

这一课新增了：

```text
app/core/config.py
```

里面定义了一个 `Settings` 类：

```python
class Settings(BaseSettings):
    app_name: str = "AI Knowledge Assistant"
    app_env: str = "development"
    app_version: str = "0.1.0"
    service_name: str = "ai-knowledge-assistant"
    database_url: str | None = None
    openai_api_key: str | None = None
    model_name: str = "gpt-4.1-mini"
```

这些字段都有默认值。

所以即使你还没有创建 `.env` 文件，项目也能正常运行。

## `.env.example` 是什么

`.env.example` 是配置模板。

它告诉你：

```text
这个项目将来需要哪些环境变量。
```

真正的 `.env` 可以放真实密钥，但不能提交到 GitHub。

一般规则是：

```text
.env.example 可以提交
.env 不要提交
```

因为 `.env` 里以后可能会有 API Key、数据库密码。

## 为什么使用 `pydantic-settings`

我们没有自己手写一堆读取环境变量的代码，而是使用：

```python
from pydantic_settings import BaseSettings
```

它可以自动完成：

```text
从 .env 读取配置
把字符串转换成合适类型
给缺失字段使用默认值
忽略暂时用不到的额外配置
```

这就是工程里常说的“不要重复造轮子”。

## 为什么有 `get_settings`

`config.py` 里还有：

```python
@lru_cache
def get_settings() -> Settings:
    return Settings()
```

大白话说：

```text
第一次读取配置时创建 Settings，以后直接复用。
```

这样不用每次请求都重新读取一遍 `.env`。

## `main.py` 发生了什么变化

以前 `main.py` 里是硬编码：

```python
app = FastAPI(
    title="AI Knowledge Assistant",
    version="0.1.0",
)
```

现在改成：

```python
settings = get_settings()

app = FastAPI(
    title=settings.app_name,
    version=settings.app_version,
)
```

也就是说：

```text
FastAPI 应用的名称和版本来自配置。
```

`/health` 也多返回了运行环境：

```json
{
  "status": "ok",
  "service": "ai-knowledge-assistant",
  "environment": "development"
}
```

## 怎么验证

运行测试：

```powershell
.\.venv\Scripts\python.exe -m pytest
```

应该看到：

```text

```

启动服务：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

访问：

```text
http://127.0.0.1:8000/health
```

## 这一课要记住什么

1. 配置管理的目的，是把“容易变化的东西”从代码里拿出去。
2. `.env.example` 是模板，`.env` 才是真实本地配置。
3. API Key 和数据库密码不要提交到 GitHub。
4. `pydantic-settings` 可以帮我们优雅读取配置。
5. 测试要跟着接口变化一起更新。
