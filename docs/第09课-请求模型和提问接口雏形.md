# 第09课：请求模型和提问接口雏形

## 这一课要解决什么

前面我们已经有了：

```text
GET /health
```

它不需要用户传任何内容，只负责告诉我们服务是否正常。

但知识库助手的核心不是“我还活着”，而是：

```text
用户问问题，系统回答问题。
```

所以这一课新增第一个提问接口：

```text
POST /questions/ask
```

现在它还不会真正调用大模型，也不会检索文档。

它先返回一个占位答案。

这一课的重点不是智能回答，而是先学会：

```text
后端怎么接收用户提交的 JSON
怎么定义请求模型
怎么定义响应模型
怎么先搭出一个可测试的接口雏形
```

## 为什么用 POST

`/health` 用的是：

```text
GET /health
```

因为它只是查询服务状态。

提问接口用的是：

```text
POST /questions/ask
```

因为用户要提交一段内容：

```json
{
  "question": "什么是 RAG？"
}
```

大白话：

```text
GET 更像去货架上拿现成东西。
POST 更像递一张纸条给后端，让后端处理。
```

提问属于后者。

## 什么是请求体

请求体就是用户发给后端的数据。

这一课里，请求体长这样：

```json
{
  "question": "What is RAG?"
}
```

它像一张纸条：

```text
我的问题是：What is RAG?
```

后端不能靠猜，所以我们定义了请求模型。

## `QuestionRequest` 是什么

在 `app/api/schemas.py` 里新增了：

```python
class QuestionRequest(BaseModel):
    question: str
```

大白话：

```text
用户提交的问题必须有 question 字段，而且它应该是字符串。
```

这就是请求模型。

它规定了前端或者调用者应该怎么给后端递纸条。

## `QuestionAnswerResponse` 是什么

我们还新增了响应模型：

```python
class QuestionAnswerResponse(BaseModel):
    question: str
    answer: str
    sources: list[str]
```

它规定后端回答时要包含：

```text
question 用户原始问题
answer   系统回答
sources  引用来源
```

现在还没有真实文档检索，所以：

```text
sources = []
```

但我们提前保留这个字段，是因为这个项目最终要做知识库问答。

知识库问答不能只给答案，还应该能说：

```text
这个答案参考了哪些资料。
```

## 新增的 `app/api/questions.py`

这一课新增：

```text
app/api/questions.py
```

它专门管理提问相关接口。

里面有：

```python
router = APIRouter(prefix="/questions", tags=["questions"])
```

这里的 `prefix="/questions"` 意思是：

```text
这个文件里的接口都以 /questions 开头。
```

然后：

```python
@router.post("/ask")
```

合起来就是：

```text
POST /questions/ask
```

## 为什么现在返回假答案

因为我们现在还没有：

```text
文档上传
文档解析
文本切分
向量检索
大模型调用
引用来源生成
```

如果现在硬接大模型，知识点会全搅在一起。

所以我们先搭接口管道：

```text
用户问题进来
后端接住
返回结构化答案
测试确认它能跑
```

后面再把 `answer` 的生成方式替换掉。

这就是工程里常见的做法：

```text
先搭骨架，再替换内部实现。
```

## 旁支小课堂：接口雏形

接口雏形不是偷懒。

它的价值是让前后端先约定好：

```text
请求地址是什么
请求字段是什么
返回字段是什么
错误时大概怎么处理
```

真实团队里，经常会先做一个假接口。

前端可以先接页面，后端可以慢慢把假逻辑换成真逻辑。

只要接口合同稳定，内部实现就可以逐步升级。

## 今天的验收标准

完成之后，你应该能做到：

1. 知道请求体是用户提交给后端的 JSON。
2. 知道 `QuestionRequest` 是提问接口的请求模型。
3. 知道 `QuestionAnswerResponse` 是提问接口的响应模型。
4. 知道 `POST /questions/ask` 是第一个提问接口雏形。
5. 知道假答案不是最终功能，而是为了先搭通接口管道。
